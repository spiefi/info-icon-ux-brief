01 — Current State
What's wrong right now

The ⓘ icon is embedded inline in the value proposition copy. On desktop this isn't a tap target issue — it's a hierarchy, discoverability, and semantic problem.

🔴 Hidden in the hero copy
Users scan value props — they don't read them character by character. A 16px grey circle at the end of a bold sentence is invisible. The eye goes: number → "Cashback" → CTA button. The ⓘ lives in no-man's land.
🔴 Wrong semantic weight
This ⓘ links to offer terms and conditions — legally required info. Mid-sentence placement signals "minor footnote." Offer terms deserve their own clearly-labelled surface, not an afterthought icon.
🔴 Visual noise in the value prop
"5% Cashback + Up to 15€ Top-Up Bonus" is the single most important line. The ⓘ creates visual clutter right where the design needs to be clean and scannable. It's adding UI where the message needs to breathe.
🔵 Desktop ≠ mobile, but the principle holds
On desktop, cursor precision means a small icon is technically clickable. But discoverability and hierarchy are platform-agnostic problems. Users don't hover random characters looking for tooltips.

04 — Recommendation
What Daria should build

For the iGraal cashback extension modal, Pattern A is the best fit. Here's the exact spec.

⭐ Primary recommendation
Pattern A — "How it works" chip → Inline dropdown
Remove the ⓘ from the value proposition text entirely. Add a clearly-labelled link element between the value card and the CTA button. On click, expand an inline details section with offer terms. The hero copy stays punchy and scannable.
Position: Below value card, above "Activate cashback" CTA — vertically centered between them
Element: Inline flex row — filled ⓘ icon (14px, brand purple) + text label "How it works" (purple, 12px, semibold)
Hover state: Underline on label text, slight opacity shift on icon — standard link affordance
Click behaviour: Expand inline dropdown below the chip — push CTA and dismiss link down. No popover/overlay (avoids z-index issues in extension frame)
Dropdown content: Title ("How Top-Up Bonus works") + bullet list (eligibility, cashback cap, credit timeline, validity) + optional "Full terms →" link
Collapse: Click the chip again to toggle closed. Also auto-collapse if user clicks "Activate cashback"
Animation: Smooth height transition (200ms ease-out) — no jarring pop
Fallback
Pattern B — if details are minimal (1–2 lines)
If the offer terms can be expressed in a single sentence ("5% cashback on top-ups ≥20€, max 15€, valid until 30 Apr"), use the card-corner ⓘ with a tooltip on hover. Simpler to build, but only works for very short content.
🧩 Nalu Component Mapping Optional — if Daria adopts Nalu
Info chip
Use NaluLink or NaluButton variant="ghost" with icon slot for the "How it works" trigger. Style with iGraal brand purple.
Dropdown panel
Use NaluCollapsible or build a simple expand/collapse with CSS transitions. Content is a NaluText block with bullet list.
Tooltip (Pattern B)
Use NaluTooltip if available, or a positioned NaluPopover triggered on hover. Dark background, light text, 8px radius.
Icon
Use NaluIcon name="info" — filled variant, not outline. 14px in the chip, 18px in the card corner (Pattern B).
05 — Design Principles Applied
Why this matters

Three UX principles at play:

👁️
Visual Hierarchy
The value proposition is the #1 element. Nothing should compete with it visually. The ⓘ icon — no matter how small — adds noise to the one line that needs to be effortless to read.
📐
Progressive Disclosure
Show the minimum needed to decide, reveal detail on demand. The hero sells the offer. The "How it works" link serves users who want more. Two layers, two purposes — don't merge them.
🎯
Fitts's Law (desktop variant)
On desktop, click accuracy is high but attention is scarce. A labelled link is faster to parse than a mystery icon. Users shouldn't have to decode what the ⓘ does — the label tells them.
