The ⓘ Trigger
Done Right

Where to place the info trigger in the iGraal extension popup — so it's findable, tappable, and doesn't mess with the hero message.

For Daria · iGraal Extension
Platform Desktop browser extension
Scope Trigger placement only — opens separate popup
Recommendation Pattern A — standalone row below value
01 — Current State
What's wrong with the trigger

The ⓘ is currently inline in the value proposition. It's a trigger for a details popup — it needs to be clearly tappable, not buried in copy.

🔴 Invisible trigger
Users scan the hero number ("7%") and go straight to the CTA. A small grey circle at the end of the sentence is in the eye's blind spot — it's not part of the reading pattern.
🔴 Pollutes the hero message
"Up to 7% cashback!" is the hook. The ⓘ adds visual noise to the single most important line. The trigger is a secondary action — it shouldn't live inside the primary message.
🔴 No affordance it's clickable
A grey outline circle doesn't signal "click me for more info." On desktop, there's no hover state visible, no label, no underline — nothing that says "this is interactive."
🔵 The trigger opens a separate popup
This ⓘ doesn't need to hold content — it just needs to be a clear, clickable trigger. That simplifies the problem: we only need to nail placement and styling.

03 — Interaction States
How the trigger should behave

Since this is desktop, hover states are critical. The trigger needs to signal "I'm clickable" before and during hover.

Default
i
How it works
Filled orange ⓘ + orange label. Clearly a separate element from the hero text.
Hover
i
How it works
Darken icon + underline label. Cursor: pointer. Optional: subtle warm background tint.
Active / Pressed
i
How it works
Further darken + slight scale-down on icon (95%). Then opens the details popup.
04 — Recommendation
What to build

Pattern A is the best fit. Here's the spec for Daria.

⭐ Build this
Pattern A — Labelled ⓘ trigger
Remove the ⓘ from the "Up to 7% cashback!" line entirely. Add a standalone trigger element between the value text and the CTA button. On click, it opens the existing details popup — no new surfaces to build.
Position: Centred, below "instead 3%", above the CTA — with ~12px spacing above and below
Layout: Inline-flex row — circle icon (20px) + text label (12px semibold) with 6px gap
Icon: Filled circle, iGraal orange (#FD5A13), white "i" — not outline, not grey
Label: "How it works" in iGraal orange, 12px, semibold (600)
Hover: Darken to #e04d0d, underline label, cursor: pointer, optional warm bg tint
Click: Opens the existing details popup — same as current ⓘ behaviour
Transition: Color and underline transitions at 150ms ease — no jarring state changes
Alternative
Pattern B — ⓘ beside "instead 3%"
If the team wants the trigger to be more subtle — icon only, no label, paired with the comparison text. More compact but less discoverable.
Fallback
Pattern C — text link, no icon
If the ⓘ icon isn't needed at all — a simple "See offer details" link does the job with zero visual overhead.
05 — Why It Matters
The principle
👁️
Separate primary from secondary
The value prop is primary — it sells. The details trigger is secondary — it informs. They shouldn't share the same line. Different jobs, different elements.
🎯
Label > mystery icon
A labelled trigger ("How it works") is faster to parse than a bare ⓘ. On desktop, screen space isn't tight enough to justify icon-only. Say what it does.
🎨
Brand orange, not grey
A grey outline circle signals "disabled" or "footnote." iGraal orange signals "this is ours, this is interactive." Use the brand — that's what it's for.
